## 우선순위 C — 파이프라인 개선 (첫 호 발행 이후)

### C-1. RSS 파싱 견고화

**✅ 완료 (2026-07-12)** — `scrape_rss.py`에 5개 항목 모두 반영: `content:encoded` 우선 사용
(+ HTML 태그 제거), `pubDate` ISO 8601 정규화, 짧은 본문(30자 미만) 스킵, 지수 백오프 재시도
(1초→2초→4초), 소스별 `[OK]/[ERROR]/[SKIP]/[WARN]` 로그. 실제 etnews 피드 + 합성 XML로 검증 완료.

```
scrape_rss.py의 RSS 파싱이 표준 RSS 2.0을 가정하고 있다.
실제 etnews RSS를 대상으로 다음을 개선해줘:

1. content:encoded 같은 네임스페이스 태그도 파싱
2. pubDate 형식을 ISO 8601로 정규화 (지금은 원문 문자열 그대로)
3. 본문이 비어있거나 너무 짧은 기사는 스킵 (태깅 API 비용 절약)
4. 네트워크 실패 시 재시도 (지수 백오프)
5. 각 소스별 수집 성공/실패를 로그로 남기기

의존성 없이 표준 라이브러리만 사용해야 한다 (CLAUDE.md 규칙).
```
