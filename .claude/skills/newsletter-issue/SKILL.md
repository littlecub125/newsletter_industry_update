# 뉴스레터 발행 준비

이 스킬은 뉴스레터 한 호를 **발행 직전 초안까지** 준비한다. `CLAUDE.md`의
"완전 자동 발행 금지" 원칙(사람이 최종 검수해야 함 — 이 서비스가 범용 AI 뉴스 요약
툴과 다른 유일한 차별점)에 따라, 실제 스티비 발송 버튼을 누르는 것은 항상 사용자
몫으로 남긴다.

## 0. 사전 확인

- `.env`의 `ANTHROPIC_API_KEY`에 크레딧이 있는지 먼저 확인한다 (`TODO.md`에 크레딧
  부족으로 태깅이 막혔던 이력이 있다 — 같은 문제가 재발했는지 실행 전에 짧게 점검).
- 이미 `data/tagged_articles.jsonl`에 이번 주 분량이 충분히 쌓여 있으면 1번(수집+태깅)을
  건너뛰고 바로 2번(초안 생성)으로 가도 된다 — 매번 새로 수집할 필요는 없다.

## 1. 수집 + 태깅 실행 (필요한 경우만)

**처음 실행이거나 리스크를 줄이고 싶으면 항상 테스트 모드부터** (`PROMPTS.md` A-2와
동일한 원칙 — API 비용과 실패 리스크를 줄이기 위함):

1. `config/sources.json`을 복사해 산업 1~2개만 남긴 임시 파일(예:
   `data/sources_test.json`)을 만든다.
2. `pipeline/scrape_rss.py`의 `collect_all(sources_path=...)`을 이 임시 파일로 호출하는
   짧은 스크립트나 REPL 한 줄로 테스트 수집을 실행하고, 수집 건수를 확인한다.
3. 문제 없으면 `python pipeline/run_pipeline.py`로 전체 수집+태깅을 실행한다. 이 스크립트는
   `config/sources.json`(전체)을 그대로 쓰므로, 전체 실행 전에 사용자에게 "전체 산업
   대상으로 실행합니다, API 비용이 발생합니다"라고 확인받는다.
4. 실행 후 `data/tagged_articles.jsonl`에서 신규 기사 수, 태깅 성공/실패 건수를 보여준다.

## 2. 태깅 결과 검수 (건너뛰지 말 것)

- 신규 태깅된 기사 중 5건 정도를 골라 제목과 태깅 결과(`industries`, `event_type`,
  `sentiment`, `impact_score`, `summary`, `reasoning`)를 나란히 보여주고 사용자 확인을
  받는다. **LLM 태깅은 초기 정확도가 낮다**(`CLAUDE.md` "태깅 품질" 참고, 특히
  `sentiment`의 "중립 vs 혼조", `controversy_flag` 판단) — 이상해 보이는 태깅이 있으면
  넘어가지 말고 짚어준다.
- `summary`가 원문을 그대로 옮기지 않고 재서술됐는지 눈으로 한 번 확인한다 (저작권
  규칙).

## 3. 티어별 이메일 초안 생성

`pipeline/build_newsletter.py --tier <티어> --week "<주차 라벨>"`을 티어별로 실행한다.

- `free`: 인자 없이 기본 실행.
- `premium`: `--industries`로 이번 호에 포함할 산업(최대 5개, `config/tiers.json` 한도
  참고)을 지정한다. 어떤 산업을 포함할지 정해진 게 없으면 사용자에게 물어본다.
- `vip_individual`/`vip_business`: `--industries` + `--companies`(고객이 지정한 회사명)를
  함께 지정한다. 이 두 티어는 아직 구독 폼에 선택 UI가 없어(`CLAUDE.md` TODO #7)
  운영자가 직접 지정하는 방식이므로, 실제 VIP 구독자가 있는 경우에만 실행한다.
- 각 실행 결과는 `data/newsletter_drafts/{week}_{tier}.html`에 저장된다. 이 HTML은
  상단에 검수 필수 배너와, 기사 사이마다 `[[ 운영자 코멘트를 여기에 추가하세요 ]]`
  자리표시가 이미 들어가 있다 — **이 자리를 대신 채우지 않는다.** 운영자 코멘트가 이
  뉴스레터의 차별점이므로 사람이 직접 쓰는 자리로 남긴다.

## 4. 마무리

- 생성된 초안 파일 경로들을 사용자에게 정리해서 보여주고, "스티비 에디터에 그대로
  복사해 붙여넣은 뒤, 코멘트 자리를 채우고, 최종 검수 후 직접 발송하시면 됩니다"라고
  안내한다.
- 이 스킬은 **스티비 발송 버튼을 대신 누르지 않는다.** 발송 여부/시점은 항상 사용자가
  결정한다.
- `TODO.md`의 "최근 완료"에 이번 호 준비 사실을 기록할지 사용자에게 확인한다.
- 발행 후 마케팅 포스팅이 필요하면 `marketing_claude`의 `instagram-post` 스킬(다른
  작업공간 소유)로 넘어가면 된다 — 이 스킬은 관여하지 않는다.
